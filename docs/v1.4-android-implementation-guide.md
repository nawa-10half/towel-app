# v1.4 ゲーミフィケーション — Android 実装ガイド

iOS版 (towel-app) の v1.4 変更点を Android 版 (towel-app-android) で再現するためのリファレンス。

## 概要

- バッジ（アチーブメント）14個: 個人9 + 家族3 + Pro限定2
- ユーザープロフィール: アイコン・カラー選択
- 交換履歴にユーザー名表示（グループモード時）
- キラキラエフェクト（「余裕あり」ステータス + 獲得バッジ）
- Game-icons.net カスタムバッジアイコン（CC BY 3.0 クレジット必須）

---

## 1. Firestore スキーマ変更

iOS版で既にデプロイ済み（Firestoreルールも更新済み）。Android版は読み書きするだけ。

### 新規フィールド（既存ドキュメント）

```
/users/{uid}
  ├── totalExchangeCount: Int          ← 交換時に +1
  ├── totalConditionCheckCount: Int    ← 診断時に +1
  ├── conditionScoreSum: Int           ← 診断時に +overallScore
  ├── conditionScoreCount: Int         ← 診断時に +1
  ├── iconName: String                 ← デフォルト "person.fill"
  ├── iconColor: String                ← デフォルト "gray"
  └── pinnedBadgeId: String?           ← 表示バッジID（null = 最高Tier自動選択）

/groups/{groupId}
  └── totalExchangeCount: Int          ← グループモード交換時に +1

/groups/{groupId}/members/{uid}
  ├── iconName: String?
  ├── iconColor: String?
  ├── pinnedBadgeId: String?
  └── exchangeCount: Int?              ← そのメンバーの交換回数
```

### 新規サブコレクション

```
/users/{uid}/achievements/{achievementId}
  ├── unlockedAt: Timestamp
  ├── seen: Bool
  └── tier: String                     ← "1"=Bronze, "2"=Silver, "3"=Gold

/groups/{groupId}/achievements/{achievementId}
  ├── unlockedAt: Timestamp
  └── unlockedBy: String (userId)
```

### ExchangeRecord 変更

```
/...towels/{towelId}/records/{recordId}
  └── createdBy: String?               ← 新規。アプリからの交換時に userId を記録
                                          Alexa 経由は createdBy なし（note で区別）
```

## 2. カウンタ加算ロジック

### 交換記録追加時（addRecord 相当）

同一バッチで以下を実行：

```
1. records サブコレクションに新規ドキュメント（createdBy: userId を含む）
2. towel ドキュメントの lastExchangedAt 更新
3. /users/{uid}.totalExchangeCount を FieldValue.increment(1)
4. グループモード時:
   - /groups/{gid}.totalExchangeCount を increment(1)
   - /groups/{gid}/members/{uid}.exchangeCount を increment(1)
```

### 状態診断保存時（saveConditionCheck 相当）

診断ドキュメント保存後：

```
/users/{uid} に merge で:
  - totalConditionCheckCount: increment(1)
  - conditionScoreSum: increment(overallScore)
  - conditionScoreCount: increment(1)
```

| ID | カテゴリ | Tier | 条件 | タイトル |
|---|---|---|---|---|
| `first_exchange` | exchange | Bronze | 交換 ≥ 1 | はじめの一歩 |
| `exchange_10` | exchange | Bronze | 交換 ≥ 10 | 交換10回 |
| `exchange_50` | exchange | Silver | 交換 ≥ 50 | 交換50回 |
| `exchange_100` | exchange | Gold | 交換 ≥ 100 | 交換100回 |

| `full_house` | condition | Silver | 全アイテム診断済み | フルハウス |
| `family_member` | group | Bronze | グループ参加中 | 家族の一員 |
| `family_builder` | group | Bronze | グループ作成者 | 家族の家主 |

## 4. 評価ロジック

### 評価タイミング

| トリガー | 評価カテゴリ |
|---|---|
| 交換記録追加後 | exchange, group |
| 状態診断完了後 | condition |
| グループ作成/参加後 | group |
| Pro購入成功後 | milestone, condition |

### 評価の流れ

1. カウンタ値（users doc から取得）と条件を比較
2. 条件を満たしたバッジが未アンロックなら Firestore に書き込み
3. 個人バッジ → `/users/{uid}/achievements/{id}`
4. 家族バッジ → `/groups/{gid}/achievements/{id}`
5. トースト通知をキューに追加

### 遡及付与（初回のみ）

`SharedPreferences` / `UserDefaults` の `achievements_migrated_v1` フラグで一度だけ実行。

1. 全タオルの records / conditionChecks をカウント
2. カウンタを users doc に書き込み（increment ではなく上書き）
3. 全バッジを評価（トーストは出さない = silent モード）

### full_house の特殊評価

カウンタではなく、メモリ上のタオルリストから `全アイテムの latestConditionCheck != nil` を判定。

## 5. プロフィール機能

### アイコン選択肢（15個）

```
person.fill, person.crop.circle.fill, face.smiling, figure.stand,
dog.fill, cat.fill, bird.fill, fish.fill, hare.fill,
tortoise.fill, lizard.fill, pawprint.fill, heart.fill, star.fill, leaf.fill
```

Android では Material Icons の対応アイコンに置き換え。

### カラー選択肢（7色）

| 名前 | 用途 |
|---|---|
| blue | プロフィールアイコン背景 |
| orange | 同上 |
| green | 同上 |
| pink | 同上 |
| purple | 同上 |
| brown | 同上 |
| gray | デフォルト |

### 同期先

プロフィール更新時に以下を同時更新：
- `/users/{uid}` — iconName, iconColor
- `/groups/{gid}/members/{uid}` — iconName, iconColor（グループ参加時のみ）

pinnedBadgeId 更新時：
- `/users/{uid}` — pinnedBadgeId
- `/groups/{gid}/members/{uid}` — pinnedBadgeId（グループ参加時のみ）

## 6. UI 構成

### アイテムリスト画面
- **左上**: プロフィールワイプ（アイコン32pt + バッジ28pt 横並び）→ タップでプロフィールSheet
- **「余裕あり」行**: アイコン上にキラキラ sparkle エフェクト（3つ、staggered、2サイクルで停止）

### プロフィール Sheet
- 大アイコン + 表示名 + バッジ数
- 最近獲得したバッジ（横スクロール、最大5個）
- 「すべてのバッジを見る」ボタン → バッジ一覧
- 「プロフィールを編集」ボタン → 編集画面

### バッジ一覧画面
- セクション: 個人 / 家族 / Pro限定
- 獲得済み: カスタムSVGアイコン + Tier色リング + キラキラエフェクト
- 未獲得: アイコンをモノクロ25%透過 + 鍵アイコンオーバーレイ
- タップで詳細（進捗バー、獲得日、バッジpin/unpin）

### プロフィール編集画面
- 表示名 + アイコングリッド + カラーグリッド
- オフラインガードあり

### 設定画面
- アカウントセクション先頭にプロフィール行 + バッジ一覧NavLink
- グループメンバーリストにアイコン + pinnedバッジ表示
- アプリ情報に「Badge icons by Game-icons.net (CC BY 3.0)」クレジット

### 交換履歴
- グループモード時: `createdBy` → メンバー名を引いて「〇〇 が交換」表示
- Alexa経由（note あり）: 従来通り note を表示

### トースト通知
- 画面上部からスライドイン、2.5秒で自動消去
- 複数同時はキューで順次表示
- 遡及付与時は出さない

## 7. バッジアイコンアセット

- ソース: Game-icons.net（CC BY 3.0）
- 形式: SVG、512x512、白アイコン + Tier色グラデーション背景（円形）
- iOS版: `Assets.xcassets/Badges/badge_{id}.imageset/`
- Android版: `res/drawable/` に SVG or VectorDrawable で配置
- **SVG にグラデーション背景が含まれているため template rendering 不要**（そのまま表示）

## 8. Firestore ルール（デプロイ済み）

```
/users/{uid}/achievements/{id}
  → allow read, write: if auth.uid == userId

/groups/{gid}/achievements/{id}
  → allow read, write: if isMember()

/groups/{gid} update
  → memberCount 不変の update も許可（カウンタ加算対応）
```

## 9. アカウント削除時のクリーンアップ

`/users/{uid}/achievements/*` サブコレクションを明示削除してから users doc を削除。
グループ achievements は共有データのため残置（既存仕様通り）。
